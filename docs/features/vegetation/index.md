# 植被分割

基于颜色植被指数的植被点提取工具。通过计算每个点的颜色植被指数值并与阈值比较，将点云分割为植被点和非植被点。支持四种植被指数和手动/Otsu 自动阈值两种模式。

## 入口路径

**菜单**: Plugins > Vegetation

**源码**: `src/plugins/vegplugin.h`

**算法**: `ct::VegetationFilter::apply()`（`libs/algorithm/vegfilter.h`）

## 前置条件

点云必须包含 RGB 颜色信息。该算法通过计算每个点的归一化颜色分量 (r, g, b) 来判断植被属性。

## 植被指数类型

| 指数 | 公式 | 说明 |
|------|------|------|
| ExG-ExR | (2g - r - b) - (1.4r - g) | Excess Green 减去 Excess Red，对绿色植被敏感 |
| ExG | 2g - r - b | Excess Green 指数，最常用的植被指数 |
| NGRDI | (g - r) / (g + r) | 归一化绿红差值指数，适合均匀光照 |
| CIVE | 0.441r - 0.811g + 0.385b + 18.787 | 颜色植被指数，对光照变化较鲁棒 |

其中 r, g, b 为归一化到 [0, 1] 的颜色分量。

## 参数说明

| 参数 | 说明 |
|------|------|
| Index Type | 植被指数类型（下拉框选择上述四种之一） |
| Threshold | 分割阈值（滑块或数值输入） |
| Otsu Auto | 是否使用 Otsu 大津法自动计算阈值 |

## 操作步骤

1. 选中含 RGB 颜色的点云
2. 打开 Plugins > Vegetation 对话框
3. 从下拉框选择 **Index Type**：
   - 一般场景推荐 **ExG** 或 **ExG-ExR**
   - 光照不均匀时推荐 **CIVE**
4. 设置阈值策略：
   - **手动**: 直接拖动滑块或输入数值，大于阈值的点被判定为植被
   - **Otsu 自动**: 勾选后自动计算最优阈值
5. 切换 Index Type 时，阈值范围和建议值会自动更新
6. 点击 **Apply** 执行分割

> **Otsu 自动阈值** 适用于植被与非植被比例相近的场景。若植被占比极小或极大，手动设置阈值效果更好。

## 输出

执行完成后生成两个点云：

- **Vegetation**: 植被点集（指数值高于阈值的点）
- **Non-Vegetation**: 非植被点集

## 适用场景

- 机载 LiDAR 配合 RGB 影像的点云
- 地面摄影测量生成的彩色点云
- 农业遥感中的作物识别
