# 法线估计与编辑

对点云进行法线估计、翻转和更新操作。法线是许多高级算法（如曲面重建、区域生长分割）的前置条件。

## 入口路径

**菜单**: Edit > Normals

**源码**: `src/edit/normals.h`

**算法**: `ct::Normals::estimate()`（`libs/algorithm/normals.h`）

## 参数说明

| 参数 | 说明 |
|------|------|
| K Search | K 近邻搜索数量，用于估计局部切平面。典型值 10~50 |
| Radius Search | 半径搜索范围（米），作为 KNN 的替代方案 |
| Viewpoint X/Y/Z | 视点坐标，用于统一法线朝向。默认使用相机位置 |
| Reverse | 是否翻转法线方向 |

## 操作步骤

### 法线估计

1. 选中点云后打开 Normals 工具
2. 设置搜索策略：
   - **K Search**: 设置邻居数量（如 20），适合密度均匀的点云
   - **Radius Search**: 设置搜索半径，适合密度变化较大的点云
3. 设置视点坐标（默认使用当前相机位置）
4. 点击 **Preview** 预览法线方向
5. 确认后：
   - **Apply**: 直接在当前点云上添加法线
   - **Add**: 创建带法线的新点云副本（前缀 `normals-`）

### 法线翻转

点击 **Reverse Normals** 将所有法线方向取反。

### 法线更新

当点云经过裁剪或滤波后，法线可能需要重新计算。点击 **Update Normals** 重新估计法线。

> 法线方向的一致性非常重要。视点坐标用于确保所有法线朝向观察者方向。若法线方向混乱，可能导致曲面重建和分割结果异常。

> 使用 K Search 和 Radius Search 二选一。K Search 适合点密度均匀的数据，Radius Search 适合密度变化的数据。不要同时设置两个参数。
