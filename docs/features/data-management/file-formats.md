# 支持的文件格式

PointWorks 支持多种业界标准的点云与网格文件格式，满足不同应用场景的数据交互需求。

## 点云格式

| 格式 | 扩展名 | 说明 | 支持的颜色 | 支持的属性 |
|------|--------|------|-----------|-----------|
| LAS | `.las` | 测绘行业标准激光雷达点云格式 | RGB | 分类、强度、回波次数、GPS 时间 |
| LAZ | `.laz` | LAS 的压缩格式（由 LASlib 支持） | RGB | 同 LAS |
| PLY | `.ply` | 斯坦福三维扫描格式，支持 ASCII/二进制 | RGB | 法线、自定义属性 |
| PCD | `.pcd` | PCL 原生点云格式 | RGB | 法线、FPFH、自定义字段 |
| E57 | `.e57` | ASTM 三维成像数据交换标准 | RGB | 强度、自定义属性 |
| TXT | `.txt`, `.xyz`, `.asc` | 文本格式，支持自定义分隔符和字段映射 | RGB（可选） | 用户自定义列 |

!!! info "TXT 字段映射"
    导入 TXT 文件时，PointWorks 会弹出字段映射对话框，允许你将文件的每一列映射到对应的点属性（X、Y、Z、R、G、B、Intensity 等）。支持自定义分隔符和跳过行数设置。

## 网格格式

| 格式 | 扩展名 | 说明 | 纹理支持 | 备注 |
|------|--------|------|---------|------|
| OBJ | `.obj` | Wavefront 三维模型格式 | 支持（自动检测 MTL/纹理） | 支持 MTL 材质文件引用 |
| STL | `.stl` | 快速原型制造标准格式 | 不支持 | 仅含几何，分 ASCII/二进制 |
| VTK | `.vtk` | VTK 传统数据格式 | 不支持 | VTK PolyData 格式 |
| IFS | `.ifs` | 索引面集格式 | 不支持 | 内部使用格式 |

## 格式选择建议

| 使用场景 | 推荐格式 | 理由 |
|----------|---------|------|
| 激光雷达点云交换 | LAS/LAZ | 行业标准，支持丰富元数据，LAZ 节省存储 |
| 航测/多源数据融合 | E57 | 支持多传感器数据、图像关联 |
| 算法处理中间结果 | PCD | PCL 原生格式，读写速度快 |
| 通用三维数据交换 | PLY | 广泛支持，格式灵活 |
| 简单坐标数据 | TXT/XYZ | 便于编辑，支持自定义列 |
| 三维模型交换 | OBJ | 支持材质和纹理，兼容性好 |
| 3D 打印 | STL | 几何信息完整，切片工具广泛支持 |

## LAS/LAZ 格式详情

LAS 格式是激光雷达点云的主流交换格式，PointWorks 通过 LASlib 库提供完整支持：

| LAS 版本 | 支持状态 | 主要特性 |
|----------|---------|---------|
| LAS 1.2 | 支持 | 点格式 0-3，基础 RGB 支持 |
| LAS 1.3 | 支持 | 点格式 0-7，扩展波形数据 |
| LAS 1.4 | 支持 | 点格式 0-10，64 位 GPS 时间、更多分类码 |

加载 LAS 文件时，PointWorks 会自动读取以下元数据：

- 点数量与包围盒
- 坐标系偏移（用于全局偏移计算）
- 点格式（确定可用属性字段）
- 分类信息

!!! tip "使用 LAZ 节省存储"
    LAZ 是 LAS 的无损压缩格式，通常可将文件大小压缩至原始大小的 10%-30%。保存时选择 `.laz` 扩展名即可自动启用压缩。
