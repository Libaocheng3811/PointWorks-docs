# 项目文件

项目文件功能允许你保存当前的工作区状态，包括已加载的点云列表、视图配置等信息，以便下次打开时快速恢复工作环境。

## 概述

PointWorks 项目文件（`.pwproj`）记录了工作区中所有点云和网格的引用关系、视图参数等状态信息。项目文件本身不存储点云数据，而是记录数据文件的路径和相关设置。

:::info[注意]
项目文件保存的是对原始数据文件的引用，而非数据本身。如果原始数据文件被移动或删除，重新打开项目时将无法找到对应的数据。

:::
## 创建项目

### 新建项目

1. 菜单栏：`文件 > 新建项目`
2. 所有已加载的数据和视图状态将被清空
3. 进入空白工作区

### 保存项目

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 保存项目 | `Ctrl + S` | 保存当前项目（首次保存会提示选择路径） |
| 另存为 | `Ctrl + Shift + S` | 将项目保存到新位置 |

项目保存时会记录以下信息：

| 记录内容 | 说明 |
|----------|------|
| 点云文件路径 | 每个已加载点云的源文件路径 |
| 全局偏移设置 | 各点云的坐标偏移值 |
| 视图状态 | 相机位置、朝向等视图参数 |
| 树视图结构 | 点云的组织结构（分组、命名等） |

## 打开项目

### 打开已有项目

1. 菜单栏：`文件 > 打开项目`
2. 选择 `.pwproj` 项目文件
3. PointWorks 将自动加载所有引用的点云文件并恢复视图状态

### 最近项目

PointWorks 会自动维护最近打开的项目列表：

- 菜单栏：`文件 > 最近项目` 可看到最近打开的项目列表
- 点击列表中的项目名称即可快速打开
- 最近列表保留最近 10 个项目

## 项目管理

### 项目文件结构

项目文件采用 JSON 格式，主要包含以下内容：

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | string | 项目文件格式版本号 |
| `clouds` | array | 已加载点云的引用列表 |
| `views` | object | 视图状态信息 |
| `settings` | object | 全局设置 |

每个点云引用包含：

| 字段 | 说明 |
|------|------|
| `path` | 点云文件的绝对或相对路径 |
| `name` | 点云在树视图中的显示名称 |
| `global_shift` | 全局坐标偏移值 |
| `visible` | 是否可见 |

:::tip[路径管理]
如果项目文件和数据文件在相同目录结构下，项目文件会使用相对路径引用数据，便于整体移动项目目录。如果数据文件被移动，可以在文本编辑器中手动修改项目文件里的路径。

:::
## 最佳实践

1. **统一目录管理**：将项目文件和相关数据文件放在同一个工作目录下，使用子目录分类管理数据
2. **及时保存**：完成重要操作后及时保存项目，避免意外丢失工作状态
3. **命名规范**：为项目文件使用有意义的名称，便于区分不同项目
4. **定期备份**：定期备份项目文件和数据目录

:::warning[数据文件位置]
修改数据文件的存放位置后，项目文件中的路径引用将失效。如需移动数据，建议连同项目文件一起移动整个工作目录。

:::
