# 精配准 - ICP

精配准 (ICP, Iterative Closest Point) 是点云配准中最经典的迭代优化算法。它通过反复执行"寻找最近点对 — 估计最优变换 — 应用变换"这一过程，逐步减小两片点云之间的对齐误差，直到收敛或达到最大迭代次数。PointWorks 基于 PCL 的 ICP 实现提供了多种变体，支持点到点、点到面等不同的距离度量方式。

## 入口路径

`配准 > 精配准 (ICP)`

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 最大对应点距离 (Max Correspondence Distance) | float | 0.05 | 超过此距离的点对不参与变换估计（单位：米）。这是控制配准精度和排除错误匹配的关键参数 |
| 最大迭代次数 (Maximum Iterations) | int | 50 | ICP 算法的最大迭代轮数。增加迭代次数可提高配准精度，但会增加计算时间 |
| 变换收敛阈值 (Transformation Epsilon) | float | 1e-8 | 连续两次迭代之间变换矩阵的差异小于此值时判定为收敛 |
| 欧式适应阈值 (Euclidean Fitness Epsilon) | float | 1e-6 | 连续两次迭代之间均方误差的变化率小于此值时判定为收敛 |
| 距离度量方式 | enum | 点到点 | `点到点`：最小化点对之间的欧氏距离；`点到面`：最小化点到目标点云局部切平面的距离 |
| 使用倒数对应 (Reciprocal) | bool | true | 勾选后要求匹配是双向的（A 的最近点是 B，且 B 的最近点是 A），可提高匹配质量 |

## 操作步骤

1. 确保两片点云已通过粗配准（中心对齐、全局配准或点对配准）获得了较好的初始对齐
2. 在点云列表中选中**两片**点云（按住 Ctrl 键多选）
3. 点击菜单 `配准 > 精配准 (ICP)`
4. 在参数对话框中设置各项参数
5. 点击**确定**开始精配准
6. 算法在后台线程中迭代执行
7. 配准完成后，输出以下结果信息：
   - 最终变换矩阵（4x4 齐次变换矩阵）
   - 配准均方误差 (Fitness Score)
   - 实际迭代次数
8. 变换后的源点云作为新对象添加到列表中

## 参数调优建议

### 最大对应点距离

这是 ICP 配准中最重要的参数：

| 场景 | 建议值 |
|------|--------|
| 精密扫描件配准 | `0.001 ~ 0.005` |
| 建筑室内扫描配准 | `0.01 ~ 0.05` |
| 地面激光扫描配准 | `0.05 ~ 0.2` |
| 机载点云配准 | `0.5 ~ 2.0` |

- 阈值过小会导致有效匹配点对不足，无法正确估计变换
- 阈值过大会引入错误的匹配点对，导致配准偏差

### 最大迭代次数

- 通常 `30 ~ 100` 次迭代即可收敛
- 如果收敛曲线显示误差仍在持续下降，可增大迭代次数

### 距离度量方式

- **点到点**：通用选择，适合大多数场景，计算速度较快
- **点到面**：精度更高，适合包含平面特征的场景（如建筑物），但需要目标点云具有法线信息

## 适用场景

- **精细配准**：在粗配准基础上进行高精度优化，使两片点云达到亚毫米级对齐
- **多站拼接**：对各站点云进行逐对精配准
- **变形监测**：对不同时期扫描的同一目标进行精确配准以检测变形

!!! warning "注意"
    ICP 算法要求两片点云在配准前已有较好的初始对齐。如果初始偏差过大（旋转超过约 30 度或平移距离过大），ICP 可能收敛到错误的局部最优解。请务必先完成粗配准后再使用 ICP。

!!! tip "技巧"
    对于大规模点云，建议先在降采样后的点云上完成 ICP 配准以获取变换矩阵，然后将该矩阵应用到原始点云上。这样可以在保证精度的前提下大幅缩短计算时间。PointWorks 支持在全局配准后自动链式执行 ICP 精配准的工作流。

!!! note "提示"
    配准完成后，建议通过以下方式验证结果质量：将两片点云用不同颜色显示并叠加，在三维视窗中旋转观察重叠区域的吻合程度；特别关注特征丰富的区域（如角点、边缘）是否精确对齐。
