# 全局配准 - IA-RANSAC

全局配准 (Global Registration) 采用基于采样一致性的自动配准方法，能够在不需要人工干预的情况下，自动求解源点云到目标点云的刚体变换（旋转 + 平移）。该方法结合了**初始对齐 (Initial Alignment)** 和 **RANSAC（随机采样一致性）** 框架，通过迭代地从两片点云中采样特征点对，估计变换矩阵，并利用 RANSAC 机制剔除错误匹配，最终找到最优的刚体变换。

## 入口路径

`配准 > 全局配准 (IA-RANSAC)`

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 采样点数 (Nr Samples) | int | 3 | 每次随机采样的点对数量。最小值为 3（确定一个刚体变换至少需要 3 对非共线点） |
| 最小特征距离 (Min Sample Distance) | float | 0.05 | 采样时两点之间的最小距离。防止采样点过于集中，提高变换估计的稳定性 |
| 最大迭代次数 (Nr Iterations) | int | 1000 | RANSAC 的最大迭代次数。值越大，找到正确变换的概率越高，但耗时越长 |
| 对应点距离阈值 (Correspondence Distance) | float | 0.5 | 判断一对匹配是否为内点的最大允许距离（单位：米） |
| 特征描述子 | enum | FPFH | 用于特征匹配的点特征描述子类型，目前支持 FPFH (Fast Point Feature Histograms) |
| 搜索半径 (Feature Radius) | float | 0.1 | 计算 FPFH 特征描述子时的搜索半径。应大于点云的平均点间距 |

## 操作步骤

1. 在点云列表中选中**两片**点云（按住 Ctrl 键多选）
2. 点击菜单 `配准 > 全局配准 (IA-RANSAC)`
3. 在参数对话框中设置各项参数
4. 点击**确定**开始配准
5. 配准过程在后台线程中执行，进度条显示当前状态
6. 配准完成后，变换后的源点云作为新对象添加到列表中，同时输出配准误差信息

## 参数调优建议

### 对应点距离阈值

这是影响配准结果最关键的参数：

- 初始尝试建议设为点云空间范围的 10% ~ 20%
- 如果配准失败（误差很大），可适当增大此阈值
- 如果配准成功但精度不够，可减小此阈值后重新运行

### 最大迭代次数

- 点云重叠区域较大且特征明显：`500 ~ 1000` 次通常足够
- 点云重叠率较低或特征不明显：可增加到 `5000 ~ 10000` 次
- 迭代次数越多，计算时间越长

### 特征搜索半径

- 应设置为点云平均点间距的 5 ~ 10 倍
- 过小会导致特征描述子信息不足，匹配不准确
- 过大会使特征丧失局部区分度

## 适用场景

- **不同站点的自动拼接**：无需人工选点，自动完成多站点云的粗对齐
- **任意姿态的初始对齐**：两片点云之间存在较大旋转和平移差异
- **ICP 精配准的前置步骤**：为后续精配准提供良好的初始变换

!!! note "提示"
    全局配准执行前会自动对两片点云计算 FPFH 特征描述子，该步骤可能需要较长时间（取决于点云大小）。建议先对两片点云进行体素降采样（体素大小建议 0.02 ~ 0.05 米），在降采样后的点云上执行全局配准，再将得到的变换矩阵应用到原始点云上。

!!! warning "注意"
    IA-RANSAC 的成功率取决于两片点云的重叠率和特征丰富程度。对于缺乏明显几何特征的场景（如平坦地面、光滑曲面），配准可能失败或得到错误结果。在这种情况下，建议使用[点对配准](point-pairs.md)进行手动对齐。
