# 大点云处理策略

PointWorks 针对大规模点云（千万至亿级）提供了多项优化策略。

## 八叉树索引

PointWorks 使用自适应八叉树结构管理点云数据：

- **空间划分**：将点云按空间位置递归划分到八叉树节点
- **按需加载**：仅加载当前视锥体内的节点数据
- **缓存机制**：首次加载时自动构建索引，后续直接读取缓存

## LOD 渲染

Level of Detail（LOD）渲染策略：

| LOD 级别 | 渲染比例 | 适用场景 |
|----------|----------|----------|
| 远距离 | 1/64 ~ 1/128 | 全局浏览 |
| 中距离 | 1/8 ~ 1/16 | 区域查看 |
| 近距离 | 全部点 | 细节查看 |

渲染时根据视点距离自动切换 LOD 级别，在保持视觉质量的同时控制渲染负载。

## 流式 I/O

对于超大文件（>2GB），PointWorks 支持流式读取：

- 每批读取约 50 万个点
- 实时显示加载进度
- 内存占用可控

!!! warning "注意"
    流式 I/O 仅支持部分格式（LAS/LAZ/PLY）。E57 等格式需要一次性加载。

## 全局坐标偏移

处理大坐标值时，PointWorks 自动应用[全局坐标偏移](global-shift.md)以避免浮点精度丢失。

## 性能建议

1. **降低采样密度**：使用体素降采样减少点数
2. **关闭法线显示**：法线渲染会显著增加 GPU 负载
3. **调整点大小**：适当减小点大小可提升渲染性能
4. **分块处理**：对超大点云先裁剪感兴趣区域再处理
